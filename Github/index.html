Comandos de git:

  1. git init: Se utiliza al iniciar un proyecto.
  2. git status: Sirve para ver el estado de las modificaciones realizados.
  3. git add . : Sirve para añadir los cambios efectuados al commit.
  4. git commit -m "Descripcion de commit" : sirve para salvar los cambios en un commit.
  5. git log:
    • git log: Permite ver el historico de modificaciones.
      • Hash: ID para ese commit
      • Branch: Permite ver el Head y la rama
      • Author
      • Fecha
      • Nombre del commit
    • git log --oneline: Permite ver el historico de modificaciones en una sola linea por commit
    • git log -p: Permite ver el historico de modificaciones de forma detallada (para salir de este modo hay que teclear ":q"
    • git log --pretty="format:%H": Ver solo los HASH de cada commit
    • Más formatos de git log
  6. git config
    • user
    • email
  7. mkdir "Nombre de carpeta": crear carpeta.
  8. git init --bare : Inicializa repositorio vacío
  9. git remote add "NombreDelServidor" "Url del sevidor"
  10. git push "NombreDelServidor" "NombreRama"
  11. git remote rename "NombreAnterior" "NombreNuevo": renombrar servidor
  12. git clone "url de servidor": clonar repositorio.
  13. git pull "nombreServidor" "nombreRama"
  14. git branch: Muestra las ramas existentes
    • Practicar con ramas
    • git branch "nombre de rama": crea una nueva rama
    • git checkout "nombre de rama": Cambiar de rama
    • git checkout -b "nombre de rama": Crear y cambiar a nueva rama.
    • git marge "nombreRama": Merge entre rama actual y "nombreRama" ((:X) Sirve para salir del editor del mensaje)
    • git rebase "nombreRama": Se crean commit de "nombreRama" en "ramaHEAD"
      • marge: Junta los trabajos y genera un merge commit.
      • rebase: Aplica los commits de otra branch en la branch actual

        • Cuando hay conflico entre 2 ramas que han sido modificadas hay que corregir en visualcode que opcion se quedara y posteriormente hacer un commit para guardar los cambios.

  15. Ctrl-Z entre versiones de git
    • git restore "nombreArchivo": restaurar version de git. (antes de agregarlas)
    • git restore --staged: desahace modificaciones despues de agregarlas pero antes de commit
    • git revert "HASH commit": crea un nuevo commit deshaciendo HASH (despues de commitearlas)
  16. Guardados temporales
    • git stash: guarda temporalmente modificaciones.
    • git stash list: listado de guardados temporales.
    • gti stash apply "numeroStash": aplica los cambios
    • git stash pop: toma la ultima modificacion del listado stash, aplica los cambios y elimina del stash.
  17. Revisar commits anteriores:
    • git checkout "hashCommit": podemos ver y manipular estados anteriores de nuestro codigo, pero para guardar los cambios primero deberemos crear una rama seguido del commit que nos guarde estos cambios
  18. git diff "hash1" .. "hashActual": volver atras y eliminar commits.
  19. TAG: Versiones que 1.0/2.0:
  20. git tag -a "nombreDTag" -m "Descripcion de tag"
  21. git push "servidor" "nombreTag"

Git status

  • Head: Estado actual de nuestro código, es decir, donde nos colocó GIT.
  • Working tree: Lugar donde los archivos realmente están siendo almacenados
  • index: Lugar donde Git almacena lo que será commiteado, es decir, la ubicación entre el working tree y el repositorio de Git en sí.
